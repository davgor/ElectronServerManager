# 009.4 — Cache or lazy-load Steam cover art

`fetchCoverArt` runs a HEAD request per server on every scan. Cache URLs in memory or load images lazily in the renderer.

**Origin:** Code review — N+1 network calls on scan.

## Acceptance criteria

- [ ] Cover art URLs cached per appId (in-memory with TTL or persistent store)
- [ ] Server scan completes without awaiting all HEAD requests (lazy load acceptable)
- [ ] Tests verify cache hit avoids duplicate fetch
- [ ] `npm test`, `npm run lint` pass
